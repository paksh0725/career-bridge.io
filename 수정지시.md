## 확인된 현재 상태

역할별 폴더와 페이지 분리는 어느 정도 되어 있다. 예를 들어 `company/company.html`, `institution/institution.html`, `admin/admin.html`은 좌측 `wt-tree`에서 각각 세부 페이지로 이동하도록 구성되어 있다.

하지만 실제 HTML 내부를 보면 다음 문제가 남아 있다.

- `company.html`, `institution.html`, `admin.html` 안에 예전 포털 스타일 CSS가 대량으로 남아 있다.
- `body class="... work-tool"`와 `wt-window`, `wt-workspace`, `wt-doc`를 사용하지만, 기존 `.sidebar`, `.topbar`, `.main`, `.content`, `.tab-bar`, `.tab-panel` 코드가 같은 파일 안에 함께 남아 있다.
- `work-tool.css`에는 `body.work-tool { overflow: hidden; }`, `.wt-workspace { overflow: hidden; }`, `.wt-left-panel { overflow-y: auto; }`, `.wt-doc { overflow: hidden; }` 구조가 있으므로 내부 스크롤 설계가 가능하지만, 실제 각 페이지 본문은 이 원칙을 일관되게 쓰지 못하고 있다.
- `company.html`, `institution.html`에는 `bottom-ad-showcase`가 `min-height:220px`, 2열 비주얼 배너 구조로 크게 들어가 있어 업무 화면 위쪽 면적을 많이 차지한다.
- `company.html`, `institution.html`에는 아직 `tab-panel`, `panel-search`, `panel-insight`, `panel-community`, `switchTab` 계열 코드가 남아 있다. 이미 `company-search.html`, `company-insight.html`, `company-community.html`처럼 페이지가 분리되어 있으므로 대시보드에 탭 패널을 계속 유지할 필요가 없다.
- `.students-grid`, `.company-grid`, `.insight-row`, `.filter-grid`, `.filter-row2`, `.comm-layout` 등 2열 이상 그리드가 많고, 일부는 업무툴 화면에서 너무 많은 카드를 한 번에 보여주려다 스크롤/가독성을 망친다.
- 검색 필터는 `.filter-box`에 있고 결과는 `.students-grid` 또는 카드 그리드로 떨어져 있어 업무용 표/목록 흐름으로 보이지 않는다.
- `admin.html`도 이미 `admin-security.html`, `admin-members.html` 등으로 페이지가 분리되어 있는데, 내부에 `section-panel`, `switchSection` 구조가 남아 있다.
- 일부 페이지는 `work-tool.css`를 제대로 링크한 구조인데, 일부 대표 대시보드 파일은 인라인 CSS가 너무 길고 중복 override가 많다.
- `work-tool.css`에 `wt-menubar`, `wt-formula`가 정의되어 있고 일부 페이지는 `wt-formula`를 사용한다. 사용자가 요구한 “상단 메뉴바/수식줄/하단 시트탭 같은 엑셀 복잡 UI 제거” 방향과 충돌할 수 있다.

## 작업 목표

이번 작업은 기능 개발이 아니라 **페이지 목업 레이아웃 정리**다.

기능, 더미 데이터, `onclick`, 모달 흐름은 최대한 유지하고, 화면 구조만 업무툴형으로 정리한다.

## 절대 유지 규칙

- 기존 `onclick` 동작을 임의로 삭제하지 않는다.
- 기존 모달 열기/닫기 흐름을 유지한다.
- 기존 더미 데이터는 가능하면 유지한다.
- 검색, 상세 보기, 메시지, 번역, 승인, 등록 흐름을 깨지 않는다.
- 학생 화면, `index`, `login`에는 work-tool 셸을 적용하지 않는다.
- `git commit`, PR은 사용자가 요청할 때만 한다.
- 계획 파일/정리 문서는 사용자 요청 없이 수정하지 않는다.

목표는 “엑셀/워드형 옛 업무 시스템 느낌이 있는 모던 내부 업무툴”이다.

적용 규칙:

- 업무 화면에서는 큰 홍보 배너보다 표/목록/KPI가 우선

## 1단계: 구조 정리

### 해야 할 일

- `company.html`, `institution.html`, `admin.html`에서 실제 사용하지 않는 예전 `.sidebar`, `.topbar`, `.main`, `.content` 기반 CSS를 정리한다.
- `work-tool.css`에서 제공하는 구조를 기준으로 통일한다.
- 대표 구조는 아래처럼 맞춘다.

```html
<body class="portal-company work-tool">
  <main class="wt-window">
    <header class="wt-titlebar">Career Bridge · 기업 담당자</header>
    <section class="wt-toolbar wt-toolbar-core">...</section>
    <section class="wt-workspace">
      <aside class="wt-left-panel">...</aside>
      <section class="wt-doc">
        <div class="wt-document-head">...</div>
        <div class="wt-doc-body">
          <div class="wt-doc-inner">...</div>
        </div>
      </section>
    </section>
  </main>
</body>
```

### 제거/축소 대상

- 기존 `.sidebar`, `.topbar`, `.main`, `.content` 실사용 구조
- `.tab-bar`, `.tab-panel`, `switchTab` 기반의 한 파일 내 업무 전환
- `.xls`처럼 보이는 불필요한 파일명 타이틀
- 상단 메뉴바 `wt-menubar`
- 수식줄처럼 보이는 `wt-formula`
- 하단 시트 탭
- 중복 네비게이션

단, 삭제 전에 해당 코드가 실제 기능 흐름에 쓰이는지 확인한다. 쓰이고 있으면 역할별 개별 페이지 링크로 치환한다.

## 2단계: 스크롤 구조 수정

현재 문제는 확대해야만 내용을 볼 수 있거나, 페이지 전체가 길어지는 구조다.

### 목표 구조

```text
body.work-tool
└─ wt-window: 100vw / 100vh, overflow hidden
   ├─ wt-titlebar: 고정
   ├─ wt-toolbar: 고정
   └─ wt-workspace: 남은 높이
      ├─ wt-left-panel: 내부 스크롤
      └─ wt-doc
         ├─ wt-document-head: 고정
         └─ wt-doc-body: 내부 스크롤
```

### 수정 지시

- 페이지 전체 스크롤 금지.
- `body`, `wt-window`, `wt-workspace`, `wt-doc`는 바깥 스크롤을 막는다.
- 실제 내용 스크롤은 `wt-doc-body` 또는 그 안의 표/목록 영역에 둔다.
- `wt-doc-body`는 반드시 `min-height:0`, `overflow:auto` 또는 내부 분할 스크롤 구조를 가져야 한다.
- 표/목록 영역에는 스크롤바가 보이도록 한다.
- 확대/줌 상태에서도 표/목록을 스크롤로 볼 수 있어야 한다.
- 더미 데이터가 적어 스크롤이 안 보이면 행을 충분히 추가한다.

## 3단계: 대시보드 배너 축소

`company.html`, `institution.html`에서 확인된 문제:

- `bottom-ad-showcase`
- `bottom-ad-inner`
- `bottom-ad-visual`
- `bottom-ad-title`
- `bottom-ad-desc`

이 구조가 `min-height:220px`이고 2열 비주얼 배너라 업무 화면 상단을 크게 차지한다.

### 수정 지시

- 기업/기관 대시보드의 큰 `bottom-ad-showcase`는 제거하거나 접을 수 있는 얇은 안내 바 형태로 축소한다.
- 배너 높이는 40~56px 수준으로 줄인다.
- 배너 안에 큰 일러스트/가짜 패널/뱃지/비주얼 영역은 제거한다.
- 대시보드 첫 화면에는 배너가 아니라 KPI, 선형 그래프, 처리 목록이 보여야 한다.
- 커뮤니티 투표 배너는 대시보드 상단에 크게 두지 말고 커뮤니티 페이지로 이동한다.

## 4단계: 탭 패널 제거 및 페이지 분리 유지

현재 `company.html`, `institution.html`에는 이미 분리 페이지가 있다.

```text
company.html
company-search.html
company-insight.html
company-community.html
company-messages.html
company-translation.html

institution.html
inst-students.html
inst-insight.html
inst-community.html
inst-messages.html
inst-translation.html
```

그런데 대시보드 파일 안에 여전히 `tab-panel`, `panel-search`, `panel-insight`, `panel-community`, `switchTab` 구조가 남아 있다.

### 수정 지시

- `company.html`은 기업 대시보드만 담당한다.
- `institution.html`은 기관 대시보드만 담당한다.
- 검색/인사이트/커뮤니티 내용은 각각의 개별 HTML에서만 유지한다.
- 대시보드 안의 `tab-bar`, `tab-panel`, `switchTab`은 제거 또는 비활성화한다.
- 기존 버튼은 `switchTab()` 대신 해당 HTML로 이동하도록 바꾼다.

예시:

```js
// 기존
switchTab('search')

// 변경
window.location.href = 'company-search.html'
```

## 5단계: 검색/필터 화면 재배치

대상:

```text
company/company-search.html
institution/inst-students.html
```

현재 확인된 스타일:

- `.filter-box`
- `.filter-grid`
- `.filter-row2`
- `.students-grid`
- `.company-grid`
- `.student-card`

### 문제

- 필터가 4열/3열 그리드로 펼쳐져 있어 확대 시 답답하다.
- 검색 결과가 카드 그리드 중심이라 업무용 검색 화면처럼 보기 어렵다.
- 필터와 결과가 한 업무 덩어리처럼 보이지 않는다.

### 수정 지시

- 검색 필터와 결과를 하나의 문서 영역 안에서 붙여 배치한다.
- 필터 영역은 상단 고정 또는 `wt-doc-head` 아래 고정 영역에 둔다.
- 필터가 많으면 “상세 필터 접기/펼치기”로 처리한다.
- 결과는 카드 그리드보다 표 또는 compact list 중심으로 바꾼다.
- 후보자/학생 상세는 `자세히 보기` 모달 또는 드로어로 유지한다.
- 결과 영역에 내부 스크롤을 둔다.

권장 구조:

```text
검색 페이지
├─ 고정 영역
│  ├─ 검색 요약
│  ├─ 주요 필터 1행
│  └─ 상세 필터 접기/펼치기
└─ 스크롤 영역
   ├─ 결과 건수/정렬
   └─ 표 또는 compact list
```

## 6단계: 그리드 남용 정리

실제 CSS에서 확인된 다중 열 구조:

- `.students-grid { grid-template-columns: repeat(3, 1fr); }`
- 큰 화면에서 `repeat(4, ...)` override
- `.filter-grid { repeat(4, 1fr); }`
- `.filter-row2 { repeat(3, 1fr) auto; }`
- `.insight-row { grid-template-columns: 1fr 1fr; }`
- `.comm-layout { grid-template-columns: 1fr 280px; }`
- `.bottom-ad-inner { grid-template-columns: minmax(0,1fr) minmax(320px,42%); }`

### 수정 지시

- 한 페이지 본문에 2열 이상 그리드를 기본값으로 쓰지 않는다.
- 검색/심사/배너/설정/단순 목록은 1열 stack으로 정리한다.
- 데이터가 많은 경우에도 표/목록 스크롤을 우선 사용한다.
- 2열이 꼭 필요하면 페이지를 분리하거나 높이 split을 사용한다.
- `comm-layout`의 우측 인기글/투표 패널은 업무 화면에서는 아래 접힘 섹션으로 옮긴다.
- `insight-row`는 2열 카드보다 1열 그래프 섹션으로 바꾼다.

## 7단계: 대시보드/인사이트 수치 표현

수치 카드만 고정값처럼 보이면 안 된다.

### 수정 지시

- KPI 카드는 작게 유지한다.
- KPI 아래에 선형 그래프를 넣는다.
- 막대 그래프만 나열하지 말고 기간 추이를 보여준다.
- 대시보드는 위쪽 고정 영역 + 아래쪽 처리 목록 구조로 정리한다.

### 기업 대시보드 수치

- Talent Pool Size
- Match Score 평균
- Shortlist Rate
- Contact Rate
- Reply Rate
- Interview Conversion Rate
- Offer Conversion Rate
- Time-to-Shortlist
- Time-to-Hire

### 기관 대시보드 수치

- 취업지원 대상자 수
- 상담 전환율
- 상담 완료율
- 문서 제출률
- 인증 완료율
- 비자 만료 D-30
- 비자 만료 D-60
- TOPIK 4급 이상 비율
- 정주 희망률

### 관리자 대시보드 수치

- 처리 대기 건수
- 인증 승인 SLA
- 평균 처리 시간
- 보안 이벤트 수
- OCR 실패율
- 민감정보 접근 건수
- 문서 다운로드 건수

## 8단계: 관리자 파일 정리

대상:

```text
admin/admin.html
admin/admin-security.html
admin/admin-members.html
admin/admin-content.html
admin/admin-partners.html
admin/admin-stats.html
admin/admin-banners.html
admin/admin-system.html
admin/admin-orgperm.html
admin/admin-translation.html
```

확인된 문제:

- `admin.html` 안에 `section-panel`, `switchSection` 구조가 남아 있다.
- 이미 관리자 세부 페이지가 분리되어 있으므로 대시보드 안에 모든 섹션을 숨겨두는 구조가 불필요하다.

### 수정 지시

- `admin.html`은 운영 대시보드만 남긴다.
- 보안/인증, 회원, 콘텐츠, 파트너, 통계, 배너, 시스템, 권한은 각 HTML에서 담당한다.
- `section-panel`, `switchSection` 의존을 제거한다.
- `wt-tree` 링크 이동 구조를 기준으로 정리한다.
- 관리자 대시보드는 “운영자/경영자/마케터” 지표를 한 화면에 과하게 다 넣지 말고, 요약 KPI + 추이 그래프 + 처리 대기 목록으로 구성한다.

## 9단계: 메시지/번역 링크 확인

이미 역할별 메시지/번역 HTML이 존재한다.

```text
company/company-messages.html
company/company-translation.html
institution/inst-messages.html
institution/inst-translation.html
admin/admin-translation.html
```

### 수정 지시

- 기업 페이지에서 번역 클릭 시 `company-translation.html`로 이동해야 한다.
- 기관 페이지에서 번역 클릭 시 `inst-translation.html`로 이동해야 한다.
- 관리자 페이지에서 번역 클릭 시 `admin-translation.html`로 이동해야 한다.
- 다른 역할의 번역/메시지 페이지로 넘어가는 링크가 있으면 모두 수정한다.
- 공유 HTML로 합치지 않는다.

## 10단계: 암호화/AI OCR은 목업 상태로만 표시

실제 암호화, 실제 OCR 엔진, 실제 AI API는 구현하지 않는다.

### 화면 표시만 추가

- 암호화 적용
- 민감정보
- 접근 제한

- 인식 실패
- 수동 확인 필요

- 계정 잠금 건수

- OCR 처리량

- OCR 실패율
- 평균 처리 시간
- 문서 유형별 인식률

## 11단계: CSS 정리 방향

현재 대표 HTML 안에는 인라인 CSS가 너무 많다.

### 수정 지시

- 공통 work-tool 스타일은 `work-tool.css`로 이동/통합한다.
- 각 페이지 HTML에는 해당 페이지 전용 보정 CSS만 최소로 남긴다.
- 예전 포털 스타일과 work-tool 스타일이 동시에 override되는 구조를 줄인다.
- `linear-gradient`는 제거한다.
- `#2563EB` 중심 색상은 `#4f7fc8` 계열로 통일한다.
- 버튼/표/필터/배지는 `work-tool.css`의 톤을 따른다.

## 작업 순서

한 번에 전면 개편하지 말고 다음 순서로 진행한다.

```text
1. work-tool.css 기준 레이아웃 규칙 정리
2. company.html 대시보드에서 탭/대형 배너 제거
3. institution.html 대시보드에서 탭/대형 배너 제거
4. admin.html에서 section-panel/switchSection 정리
5. company-search.html 검색 필터+결과 구조 정리
6. inst-students.html 검색 필터+결과 구조 정리
7. company-insight.html / inst-insight.html 그래프 추이형으로 정리
8. admin 세부 페이지 내부 스크롤/표 구조 정리
9. messages/translation 링크 검수
10. 전체 줌 테스트 및 스크롤 테스트
```

## 최종 검수 체크리스트

- [ ] `company.html`은 기업 대시보드만 담당한다.
- [ ] `institution.html`은 기관 대시보드만 담당한다.
- [ ] `admin.html`은 관리자 운영 대시보드만 담당한다.
- [ ] 검색/인사이트/커뮤니티/메시지/번역은 개별 HTML로 이동한다.
- [ ] `tab-panel`, `switchTab`, `section-panel`, `switchSection` 의존이 불필요하게 남아 있지 않다.
- [ ] 큰 `bottom-ad-showcase`가 업무 화면 상단을 차지하지 않는다.
- [ ] 페이지 전체 스크롤이 아니라 내부 영역 스크롤로 동작한다.
- [ ] 확대/줌 상태에서도 표/목록을 볼 수 있다.
- [ ] 검색 필터와 결과가 한 덩어리로 붙어 있다.
- [ ] 표/목록 영역에 스크롤바가 보인다.
- [ ] 불필요한 2열/3열/4열 카드 그리드가 줄었다.
- [ ] 대시보드/인사이트에 선형 추이 그래프가 포함되어 있다.
- [ ] 역할별 번역/메시지 링크가 서로 섞이지 않는다.
- [ ] 학생, index, login에는 work-tool 셸이 적용되지 않았다.
- [ ] 기존 기능, 더미 데이터, `onclick`, 모달 흐름이 유지되었다.
- [ ] 실제 암호화/OCR 구현을 시도하지 않고 목업 상태값만 표시한다.
